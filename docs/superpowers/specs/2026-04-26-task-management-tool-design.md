# Task Management Tool — Design Spec

## Overview

A task management application built with the T3 stack (Next.js, TypeScript, Tailwind CSS, tRPC, NextAuth.js, Prisma) deployed serverlessly on AWS via SST v3, using Supabase as the PostgreSQL database.

## Tech Stack

- **Frontend**: Next.js (Pages Router), TypeScript, Tailwind CSS
- **API**: tRPC (end-to-end type-safe)
- **Auth**: NextAuth.js with Credentials provider (email/password), JWT sessions
- **ORM**: Prisma
- **Database**: Supabase (PostgreSQL)
- **Deployment**: SST v3 (Ion) on AWS (Lambda + CloudFront + S3)
- **Testing**: Vitest + React Testing Library
- **Scaffolding**: `npm create t3-app@7.37.0`

### Task ↔ Tag
Implicit Prisma many-to-many relation (auto-generated `_TaskToTag` join table).

### NextAuth Tables
Auto-generated by Prisma adapter: Account, Session, VerificationToken. Session table exists for adapter compatibility but JWT strategy is used at runtime.

## tRPC API

### Routers

```
src/server/api/routers/
├── auth.ts       # signup mutation
├── project.ts    # CRUD projects, invite/remove members
├── task.ts       # CRUD tasks, filter, assign, status updates
├── tag.ts        # CRUD tags per project
└── user.ts       # Get/update profile
```

### Procedures

**auth router:**
- `signup` (mutation) — Create user with bcrypt-hashed password

**project router:**
- `create` (mutation) — Create project, auto-add owner as ProjectMember(OWNER)
- `list` (query) — List projects where current user is a member
- `getById` (query) — Get project with members (membership check)
- `update` (mutation) — Update name/description (owner only)
- `delete` (mutation) — Delete project and cascade (owner only)
- `inviteMember` (mutation) — Add user by email (owner only)
- `removeMember` (mutation) — Remove member (owner only)

**task router:**
- `create` (mutation) — Create task in project (membership check)
- `list` (query) — List tasks with filters: status, priority, assignee, tag
- `getById` (query) — Get task with tags and assignee
- `update` (mutation) — Update any task field (membership check)
- `delete` (mutation) — Delete task (creator or project owner)

**tag router:**
- `create` (mutation) — Create tag in project
- `list` (query) — List tags for a project
- `delete` (mutation) — Delete tag (project owner)

**user router:**
- `getProfile` (query) — Get current user's profile
- `updateProfile` (mutation) — Update name, image

### Authorization

- `protectedProcedure` middleware ensures authenticated session
- Project-scoped procedures verify the user is a member of the project before allowing access
- Owner-only operations (delete project, invite/remove members) check `ProjectMember.role === OWNER`

## Authentication

1. **Signup**: `/auth/signup` → `auth.signup` tRPC mutation → bcrypt hash → create User via Prisma → redirect to signin
2. **Signin**: `/auth/signin` → `signIn("credentials", { email, password })` → Credentials provider verifies hash → JWT session created
3. **Session**: JWT strategy (required for Credentials provider). User id and email encoded in token.
4. **Route protection**: `getServerSideProps` checks session, redirects to `/auth/signin` if absent
5. **API protection**: `protectedProcedure` middleware in tRPC

## Dashboard

Central overview page showing:
- **Summary stats**: Total tasks assigned to me, tasks due today, tasks due this week, completed tasks
- **My tasks list**: Tasks assigned to current user across all projects, sorted by deadline
- **Recent activity**: Recently created/updated tasks across user's projects

## Deployment (SST v3 on AWS)

### sst.config.ts
- `NextjsSite` construct deploys the Next.js app via OpenNext
- Environment variables bound: `DATABASE_URL`, `NEXTAUTH_SECRET`, `NEXTAUTH_URL`

### AWS Resources
- **Lambda**: Next.js server functions
- **CloudFront**: CDN for static assets and server routing
- **S3**: Static assets bucket

### External Services
- **Supabase**: PostgreSQL database (managed). Prisma connects via connection pooler URL (port 6543, `?pgbouncer=true`).

### Environment Variables
| Variable        | Description                              |
|-----------------|------------------------------------------|
| DATABASE_URL    | Supabase pooled connection string        |
| DIRECT_URL      | Supabase direct connection (migrations)  |
| NEXTAUTH_SECRET | Random secret for JWT signing            |
| NEXTAUTH_URL    | Deployed app URL                         |

### Deployment Commands
```bash
# Local development
npm run dev

# Deploy to AWS
npx sst deploy --stage production

# Run Prisma migrations against Supabase
npx prisma db push
```

## Testing

**Framework**: Vitest + React Testing Library

**Scope** (~6-8 tests):
- **tRPC router tests** (3-4): Create task, update task status, project membership authorization
- **Utility tests** (1-2): Password hashing/verification, data formatting
- **Component tests** (1-2): Task card renders correctly, project list renders

**Test infrastructure**:
- Mock Prisma client via `vitest-mock-extended`
- Mock next-auth session for protected procedure tests
- No E2E or live database integration tests
