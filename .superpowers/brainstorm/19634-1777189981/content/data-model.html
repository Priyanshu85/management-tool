Prisma models backed by Supabase PostgreSQL

Entity Relationship Diagram

Key Relationships

  • User → owns many Projects (via ownerId)
  • User ↔ Project via ProjectMember (invite-only access)
  • Project → has many Tasks
  • Task → assigned to one User (optional)
  • Task ↔ Tag many-to-many (implicit Prisma relation)
  • Tag scoped to Project (each project has its own tags)
